# Beta User Recruitment & Feedback Plan

## Target: 50-100 Beta Users by Dec 13

### Recruitment Channels

#### 1. Direct Outreach (20-30 users)
- Email existing contacts
- LinkedIn direct messages to HR managers / CTOs in India + US
- Slack communities (HR, startups)
- **Message template:** "We're launching Targetym (all-in-one people management). Early access + 3 months free. Interested?"

#### 2. ProductHunt (20-50 users)
- Launch Dec 13 at 9am PST
- Offer free Pro for 3 months to top commenters
- Respond to all comments within 30 min
- Engage authentically

#### 3. Content Communities (10-20 users)
- HackerNews Show HN thread
- Reddit: r/startups, r/entrepreneur
- LinkedIn posts
- Twitter/X engagement

#### 4. Partnerships (5-10 users)
- Accounting software integrations
- HR tool partnerships
- Business communities

### Beta User Survey

```
1. Company size? (1-10, 11-50, 50+)
2. Which feature interests most? (Goals, Recruitment, Performance)
3. What's your biggest people management pain?
4. How likely to recommend? (1-10)
5. Any bugs or issues encountered?
6. Feature requests?
7. Willing to do 15min call?
```

### Feedback Collection Methods

1. **In-app feedback form** (simple widget)
2. **Email surveys** (sent weekly)
3. **1:1 calls** (top 10 users - 15min each)
4. **Support tickets** (monitor for patterns)
5. **Analytics** (track feature usage, drop-off points)

### Feedback Integration (14-day sprint)

Daily standup review:
- New bug reports
- Feature requests
- Support tickets
- Usage analytics
- NPS trending

Priority fixes: Deploy daily fixes for P1 bugs. Everything else goes to post-launch roadmap.

### Post-Launch (Month 1-2)

- Weekly user interviews (5 users)
- Monthly cohort analysis
- Churn investigation for any users > 5% monthly
- Refine onboarding based on feedback
- Iterate on core features
